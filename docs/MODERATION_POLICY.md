**Статус:** Документ-источник истины. Обязателен для исполнения.
**Версия:** 1.1
**Дата:** 2026-05-18
**Применяется к:** backend, WebSocket, DB schema, UI модалов модерации.

Любое изменение логики модерации в коде должно быть согласовано с этим документом.

## 1. Иерархия ролей

### 1.1 Глобальные роли

Хранятся в `users.global_role`. Действуют на уровне всей системы.

```
Уровень   Идентификатор      Назначение
──────────────────────────────────────────────────────────────
  7       root_owner         Зарезервировано. Super-owner концепт.
                             Единственный кто может модерировать
                             platform_owner. Не реализован в текущей
                             версии. Не должен создаваться через UI.
                             Зафиксирован как будущая системная роль.
  6       platform_owner     Владелец платформы. Единственный.
                             Не может быть создан через UI.
                             Не подчиняется никому из обычной
                             административной иерархии.
                             Единственный кто может применить санкцию
                             к другому platform_owner — root_owner
                             (когда будет реализован).
  5       admin              Глобальный администратор.
  4       moderator          Глобальный модератор.
  1       user               Обычный участник. По умолчанию.
```

Уровни 2 и 3 отсутствуют в глобальном пространстве намеренно —
они зарезервированы за локальными ролями (см. 1.2) во избежание коллизий.

Уровень 7 (`root_owner`) зарезервирован в схеме иерархии, но **не реализован**
в текущей версии и не существует в БД. При реализации — только через
прямую запись в БД или CLI, никогда через UI.

### 1.2 Локальные роли

Хранятся в `room_members.room_role`. Действуют в рамках конкретной комнаты.

```
Уровень   Идентификатор      Назначение
──────────────────────────────────────────────────────────────
  3       owner              Создатель или назначенный владелец комнаты.
  2       local_admin        Локальный администратор комнаты.
  1       local_moderator    Локальный модератор комнаты.
  0       member             Участник без дополнительных прав.
 —        banned             Запись о бане. Не является активной ролью.
```

Значение `banned` в `room_role` означает заблокированный доступ в комнату.
Это не роль участника, а маркер ограничения. Пользователь с `room_role = 'banned'`
не учитывается в счётчике участников и не может войти в комнату.

### 1.3 Числовая шкала уровней (единая для всех проверок)

```
  7   root_owner         (зарезервировано, не реализовано)
  6   platform_owner
  5   admin
  4   moderator
  3   owner              (локальный)
  2   local_admin        (локальный)
  1   local_moderator    (локальный)
  0   member
 -1   не состоит в комнате
```

Уровень 7 зарезервирован. В текущих проверках кода используется диапазон
–1..6. При добавлении root_owner проверки будут расширены до 7.

## 2. Приоритет глобальных и локальных ролей

### 2.1 Правило effective_level

Для любой проверки прав в контексте конкретной комнаты вычисляется один
итоговый уровень:

```
effective_level = max(global_level, local_level)
```

**Пример:**
- `global_role = 'moderator'` (уровень 4) + `room_role = 'owner'` (уровень 3)
  → `effective_level = 4`. Действует как global_mod.
- `global_role = 'user'` (уровень 1) + `room_role = 'owner'` (уровень 3)
  → `effective_level = 3`. Действует как owner комнаты.
- `global_role = 'admin'` (уровень 5) + нет записи в room_members
  → `effective_level = 5`. Глобальный уровень не требует членства.

### 2.2 Область действия

| Контекст действия | local_mod / local_admin / owner | global_mod / global_admin / owner |
|---|---|---|
| Комната, где назначена роль | ✅ | ✅ |
| Любая другая комната | ❌ | ✅ |
| Вся система (global ban) | ❌ | ✅ |

Локальные роли **строго ограничены одной комнатой**.
Глобальный модератор не имеет локальных прав owner: он не может переименовать
или удалить чужую комнату. Глобальные права распространяются только на
действия с пользователями (kick, ban, mute), но не на управление комнатой.

### 2.3 Глобальный admin и владелец комнаты

Глобальный admin (уровень 5) может применять модерационные действия к владельцу
публичной комнаты (уровень 3), так как 5 > 3.
Исключение: удалить или переименовать постоянную (permanent) комнату без
явного разрешения platform_owner — нельзя.

## 3. Матрица полномочий: кто кого может модерировать

### Условные обозначения

```
✅   разрешено
❌   запрещено
⏱️   разрешено с ограничением по максимальному сроку
🔒   разрешено только в своей комнате
📋   требуется обязательная причина
—    неприменимо
```

### 3.1 Kick

Kick — это принудительное завершение текущей сессии и WS-соединения.
Долгосрочного запрета не создаёт. Пользователь может войти снова немедленно.

| Actor ↓ \ Target → | member | local_mod | local_admin | room_owner | global_mod | global_admin | platform_owner | root_owner |
|---|---|---|---|---|---|---|---|---|
| **local_mod** | 🔒 | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **local_admin** | 🔒 | 🔒 | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **global_mod** | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ | ❌ | ❌ |
| **global_admin** | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ | ❌ |
| **platform_owner** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| **root_owner** *(резерв)* | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ |

`global_mod` может кикнуть: member, local_mod, local_admin, room_owner во всех комнатах.
`global_mod` **не может** кикнуть: global_admin, platform_owner, root_owner.
`global_admin` может кикнуть: всех кроме platform_owner и root_owner.
`platform_owner` не может быть кикнут никем из текущей иерархии (только root_owner).

### 3.2 Mute (кляп в комнате)

Mute — временная блокировка отправки сообщений в конкретной комнате.
Сессия и WS-соединение **не** завершаются.

| Actor ↓ \ Target → | member | local_mod | local_admin | room_owner | global_mod | global_admin | platform_owner | root_owner |
|---|---|---|---|---|---|---|---|---|
| **local_mod** | 🔒 ⏱️ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **local_admin** | 🔒 ⏱️ | 🔒 ⏱️ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **global_mod** | ✅ ⏱️ | ✅ ⏱️ | ✅ ⏱️ | ✅ ⏱️ | ❌ | ❌ | ❌ | ❌ |
| **global_admin** | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ | ❌ |
| **platform_owner** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| **root_owner** *(резерв)* | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ |

`global_mod` может замьютить: member, local_mod, local_admin, room_owner во всех комнатах.
`global_mod` **не может** замьютить: global_admin, platform_owner, root_owner.

### 3.3 Room ban (локальный бан в комнате)

Room ban блокирует вход в конкретную комнату. Пользователь может войти
в другие комнаты. Сессия завершается.

| Actor ↓ \ Target → | member | local_mod | local_admin | room_owner | global_mod | global_admin | platform_owner | root_owner |
|---|---|---|---|---|---|---|---|---|
| **local_mod** | 🔒 ⏱️ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **local_admin** | 🔒 | 🔒 | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **global_mod** | ✅ ⏱️ | ✅ ⏱️ | ✅ ⏱️ | ✅ ⏱️ | ❌ | ❌ | ❌ | ❌ |
| **global_admin** | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ | ❌ |
| **platform_owner** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| **root_owner** *(резерв)* | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ |

`global_mod` может забанить в комнате: member, local_mod, local_admin, room_owner
во всех комнатах, но не более 24 часов.
`global_mod` **не может** забанить в комнате: global_admin, platform_owner, root_owner.
`global_admin` может забанить в комнате любого кроме platform_owner и root_owner,
на любой срок включая permanent.

### 3.4 Global ban (глобальный бан)

Global ban полностью запрещает вход в систему. Сессия завершается.

| Actor ↓ \ Target → | member | local_mod | local_admin | room_owner | global_mod | global_admin | platform_owner | root_owner |
|---|---|---|---|---|---|---|---|---|
| **local_mod** | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **local_admin** | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **global_mod** | ✅ ⏱️ | ✅ ⏱️ | ✅ ⏱️ | ✅ ⏱️ | ❌ | ❌ | ❌ | ❌ |
| **global_admin** | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ | ❌ |
| **platform_owner** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| **root_owner** *(резерв)* | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ |

`global_mod` может глобально забанить: member, local_mod, local_admin, room_owner,
но не более 24 часов и без permanent.
`global_mod` **не может** глобально забанить: global_admin, platform_owner, root_owner.
`global_admin` может глобально забанить любого кроме platform_owner и root_owner,
на любой срок включая permanent.
`platform_owner` не может быть глобально забанен никем из обычной иерархии.

## 4. Кто кого НЕ может модерировать

Это явные запреты. Любой код, нарушающий эти правила — баг.

### 4.1 Абсолютные запреты (не зависят от контекста)

1. **Нельзя применить действие к самому себе.** Независимо от роли.
2. **Нельзя применить действие к пользователю равного или высшего уровня.**
   `effective_level(actor) > effective_level(target)` — обязательное условие.
3. **platform_owner не может быть кикнут, забанен или замьючен никем из обычной
   административной иерархии** (local_mod, local_admin, global_mod, global_admin).
   Единственное исключение — root_owner, когда будет реализован.
4. **root_owner (уровень 7, резерв) — единственная роль выше platform_owner.**
   Любое действие против platform_owner возможно только со стороны root_owner.
   До реализации root_owner — platform_owner абсолютно неуязвим для модерации.
5. **local_mod и local_admin не могут выполнять глобальный бан.**
6. **local_mod и local_admin не могут модерировать пользователей в чужих комнатах.**

### 4.2 Запреты по области действия

7. **local_mod не может модерировать local_admin, room_owner и выше**
   (даже в своей комнате).
8. **local_admin не может модерировать room_owner и выше** (даже в своей комнате).
9. **global_mod не может модерировать global_admin, platform_owner, root_owner.**
   `global_mod` ограничен уровнем < 5: может работать с member, local_mod,
   local_admin и room_owner во всех комнатах, но не с глобальными ролями выше.
10. **global_admin не может модерировать platform_owner и root_owner.**
    `global_admin` ограничен уровнем < 6: может работать со всеми ниже уровня 6.
11. **platform_owner не может модерировать другого platform_owner.**
    Самомодерация запрещена инвариантом I-1. Модерация равного — инвариантом I-2.

### 4.3 Запреты по сроку

12. **local_mod не может выдавать permanent mute или room ban.**
13. **global_mod не может выдавать permanent mute, room ban или global ban.**
14. **Любой актор кроме global_admin и platform_owner не может выдавать
    global ban срок > 24ч.**

### 5.4 Допустимые варианты срока

Фиксированный список вариантов, принимаемых системой:

```
1 час       = 1
24 часа     = 24
7 дней      = 168
30 дней     = 720
permanent   = NULL (поле expires_at равно NULL, флаг permanent = true)
```

Произвольный срок в часах не принимается. Сервер проверяет значение по списку.
UI обязан предоставить только эти варианты. Сервер обязан отклонить иное значение.

### 5.5 Kick

Kick не имеет срока. Это единоразовое действие без долгосрочного эффекта.

## 6. Что происходит с активной сессией

### 6.1 Kick

```
Действия:
  1. DELETE FROM sessions WHERE user_id = target    — все сессии удаляются
  2. WS: sendToUser(target, {event: 'force_logout', reason: 'kicked'})
  3. WS: conn->close() для каждого активного соединения цели

Состояние после:
  - Текущая сессия прекращена
  - WS соединение закрыто
  - room_members: запись без изменений (или удалена если это было единственное членство)
  - is_banned: не изменяется
  - Пользователь может зайти снова немедленно при следующем логине
  - При следующем logine — новая сессия создаётся штатно
```

### 6.2 Room ban

```
Действия:
  1. room_members SET room_role = 'banned', banned_at = NOW(), banned_by = actor_id,
                      banned_until = <expires_at или NULL>, ban_reason = <reason>,
                      ban_issued_by_role = <actor_role>
  2. DELETE FROM sessions WHERE user_id = target    — все сессии удаляются
  3. WS: sendToUser(target, {event: 'force_logout', reason: 'banned_room', room_id: X})
  4. WS: conn->close() для каждого активного соединения цели

Состояние после:
  - Текущая сессия прекращена
  - WS соединение закрыто
  - При следующем логине: вход в систему разрешён (is_banned = 0)
  - При попытке join_room в забаненную комнату: сервер отклоняет
  - Скрытие комнаты из списка в UI — допустимо, но не является защитой
  - По истечении banned_until: bан снимается автоматически при следующей попытке join
```

### 6.3 Global ban

```
Действия:
  1. users SET is_banned = 1, banned_at = NOW(), banned_by = actor_id,
                banned_until = <expires_at или NULL>, ban_reason = <reason>,
                ban_issued_by_role = <actor_role>
  2. DELETE FROM sessions WHERE user_id = target    — все сессии удаляются
  3. WS: sendToUser(target, {event: 'force_logout', reason: 'banned_global'})
  4. WS: conn->close() для каждого активного соединения цели

Состояние после:
  - Текущая сессия прекращена
  - WS соединение закрыто
  - При попытке логина: LoginHandler проверяет is_banned → отклоняет
  - Session::validate проверяет is_banned И banned_until → отклоняет до истечения
  - По истечении banned_until: is_banned автоматически переводится в 0
    в момент следующей проверки Session::validate (lazy expiry)
    либо принудительно при открытии раздела «Баны» в admin panel
```

### 6.4 Mute

```
Действия:
  1. room_members SET muted_until = <expires_at>, mute_reason = <reason>,
                      mute_issued_by_role = <actor_role>
  2. WS: sendToUser(target, {event: 'muted_in_room', room_id: X,
                              muted_until: <ISO UTC>, reason: <reason>})

Состояние после:
  - Сессия НЕ прекращается
  - WS соединение НЕ закрывается
  - Пользователь остаётся в комнате, видит сообщения
  - Пользователь НЕ может отправлять сообщения до истечения muted_until
  - MessageController проверяет muted_until при каждом send_message
  - По истечении muted_until: ограничение снимается автоматически
    при следующей проверке MessageController (lazy expiry)
```

## 7. Правила снятия санкций

### 7.1 Принцип "строго ниже"

Снять санкцию может только тот, чей уровень **строго выше** уровня актора,
наложившего санкцию.

```
Уровень актора снятия > Уровень актора наложения
```

Это правило применяется через поле `ban_issued_by_role` / `mute_issued_by_role`.

### 7.2 Матрица снятия room ban и mute

| Кто снимает ↓ \ Кто наложил → | local_mod | local_admin | global_mod | global_admin | platform_owner | root_owner |
|---|---|---|---|---|---|---|
| **local_mod** | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **local_admin** (своя комната) | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **global_mod** | ✅ | ✅ | ❌ | ❌ | ❌ | ❌ |
| **global_admin** | ✅ | ✅ | ✅ | ❌ | ❌ | ❌ |
| **platform_owner** | ✅ | ✅ | ✅ | ✅ | — | ❌ |
| **root_owner** *(резерв)* | ✅ | ✅ | ✅ | ✅ | ✅ | — |

Примечание: `—` означает неприменимо (self-revoke). `root_owner` не реализован,
строка зарезервирована для будущей реализации.

### 7.3 Матрица снятия global ban

| Кто снимает ↓ \ Кто наложил → | global_mod | global_admin | platform_owner | root_owner |
|---|---|---|---|---|
| **global_mod** | ❌ | ❌ | ❌ | ❌ |
| **global_admin** | ✅ | ❌ | ❌ | ❌ |
| **platform_owner** | ✅ | ✅ | — | ❌ |
| **root_owner** *(резерв)* | ✅ | ✅ | ✅ | — |

### 7.4 Обязательная причина снятия

Снятие любой санкции требует обязательного поля `revoke_reason`.
Причина не может быть пустой строкой.

Это требование применяется:
- при снятии room ban
- при снятии global ban
- при снятии mute

**Исключение:** автоматическое истечение срока (система) — причина не требуется.
Записывается как `revoke_reason = 'expired'` системно.

### 7.5 При снятии бана — сессия не восстанавливается

Снятие бана не создаёт новую сессию и не отправляет WS-событие пользователю.
Пользователь может войти заново при следующем логине.

---

## 8. Политика IP/email ban

### 8.1 Текущий статус

IP-бан и email-бан **не реализованы** в текущей версии.

### 8.2 Планируемое поведение (не реализовывать без явного решения)

Когда будет реализовано:

- IP-бан: блокировка по IPv4/IPv6 адресу. Проверяется при `Session::create()`.
- Email-бан: блокировка по email-адресу. Проверяется при `RegisterHandler` и `LoginHandler`.

### 8.3 Доступность расширенных данных пользователя

IP адрес, email, user-agent доступны только авторизованному персоналу:

| Данные | Кто видит |
|---|---|
| IP адрес (из сессии) | local_mod/local_admin (только своя комната), global_mod, global_admin, platform_owner |
| Email | local_mod/local_admin (только своя комната), global_mod, global_admin, platform_owner |
| User-Agent | local_mod/local_admin (только своя комната), global_mod, global_admin, platform_owner |

Для просмотра данных пользователя модератор должен быть членом комнаты (для локальных)
или иметь глобальную роль moderator и выше.

## 9. Инварианты

Это неизменяемые правила. Любой код, нарушающий хотя бы один из них — некорректен.

### I-1: Нельзя применить действие к себе

```
actor_id != target_id  — всегда, без исключений
```

### I-2: Уровень актора должен быть строго выше уровня цели

```
effective_level(actor, room_id) > effective_level(target, room_id)
```

Для глобальных действий (global ban):
```
global_level(actor) > effective_level(target, any_room)
```

### I-3: platform_owner не подлежит модерации со стороны обычной иерархии

```
target.global_role == 'platform_owner'
  AND actor.global_role != 'root_owner'
  →  все действия запрещены
```

Единственное исключение — `root_owner` (уровень 7).
До реализации `root_owner` условие `actor.global_role != 'root_owner'` всегда
выполняется, то есть `platform_owner` абсолютно неуязвим для модерации.

### I-4: Срок санкции не может превышать максимум для роли актора

```
actor_role → allowed_durations (см. раздел 5)
requested_duration NOT IN allowed_durations  →  ошибка
```

Сервер обязан проверять это независимо от данных клиента.

### I-5: Причина обязательна для ban и mute

```
action IN ('ban_room', 'ban_global', 'mute')  AND  reason == ''  →  ошибка
```

Причина может быть опциональна только для kick.

### I-6: Причина обязательна для снятия санкции

```
action IN ('unban_room', 'unban_global', 'unmute')  AND  revoke_reason == ''  →  ошибка
```

### I-7: local_mod и local_admin не выходят за пределы своей комнаты

```
actor.effective_level < 4  AND  action.room_id != actor.local_room_id  →  ошибка
```

Глобальные действия (global_ban) не существуют для этих ролей.

### I-13: global_mod ограничен целями уровня < 5

```
actor.global_role == 'moderator'
  AND effective_level(target) >= 5
  →  запрещено
```

`global_mod` может модерировать: member (0), local_mod (1), local_admin (2),
room_owner (3). Не может: global_admin (5), platform_owner (6), root_owner (7).

### I-14: global_admin ограничен целями уровня < 6

```
actor.global_role == 'admin'
  AND effective_level(target) >= 6
  →  запрещено
```

`global_admin` может модерировать всех кроме platform_owner (6) и root_owner (7).

### I-15: root_owner — зарезервированная роль, не реализована

```
users.global_role NOT IN ('root_owner')  — в текущей реализации
```

До явного решения о реализации `root_owner` его нельзя создавать, назначать
или проверять в коде. Зафиксирован только в документе как концепт.

### I-8: Снятие санкции — строго ниже

```
level(снимающий) > level(наложившего)
```

Определяется через поле `ban_issued_by_role` / `mute_issued_by_role`.

### I-9: При kick и ban — сессия и WS всегда завершаются

```
action IN ('kick', 'ban_room', 'ban_global')
  →  DELETE sessions WHERE user_id = target
  →  conn->close() для всех соединений target
```

Порядок: сначала WS-событие (force_logout), затем закрытие соединения.
Это даёт клиенту возможность получить причину до разрыва.

### I-10: Mute не завершает сессию

```
action == 'mute'
  →  сессия не трогается
  →  WS соединение не закрывается
  →  только DB write + WS-уведомление цели
```

### I-11: Автоматическое истечение срока — lazy

Срок бана и мьюта проверяется в момент доступа:
- `Session::validate` — проверяет `banned_until` при каждом HTTP-запросе и WS-подключении
- `EventRouter::onJoinRoom` — проверяет `room_members.banned_until`
- `MessageController::send` — проверяет `room_members.muted_until`

Активный принудительный сброс (cron/timer) — не обязателен.
Авто-снятие записывает `revoke_reason = 'expired'` в audit log.

### I-12: Effective level определяется в момент действия

Роль актора берётся из текущей сессии в момент применения действия.
Если роль была понижена после подключения — используется роль из сессии WS
(сессия обновляется при reconnect). Это известное ограничение архитектуры.

## 10. Определения терминов

| Термин | Определение |
|---|---|
| **Kick** | Принудительное завершение текущей сессии и WS-соединения. Без долгосрочного запрета. |
| **Room ban** | Запрет на вход в конкретную комнату на заданный срок или навсегда. Сессия завершается. |
| **Global ban** | Запрет на вход в систему на заданный срок или навсегда. Сессия завершается. |
| **Mute** | Запрет на отправку сообщений в конкретной комнате на заданный срок. Сессия и WS остаются. |
| **Permanent** | `expires_at = NULL`, `duration_hours = NULL`. Не истекает автоматически. |
| **Effective level** | Итоговый уровень прав актора в контексте комнаты: `max(global_level, local_level)`. |
| **actor_role** | Снимок роли актора в момент применения санкции. Хранится в audit log и в поле `_issued_by_role`. |
| **force_logout** | WS-событие отправляемое цели перед закрытием соединения. Содержит причину. |
| **Lazy expiry** | Срок проверяется в момент доступа, не по таймеру. |
| **root_owner** | Зарезервированная системная роль уровня 7. Не реализована. Единственная роль выше platform_owner. |
